## Pitfalls

### 坑 1: "满意" ≠ LGTM，不等于 commit

- **现象**: 上一个 session 结束时说了"满意"但没有 commit，29 个文件跨 session 遗留
- **正确做法**: 满意时应明确说"LGTM"或"commit 吧"，触发即时 commit 流程
- **根因**: 情感确认和操作确认是两个不同的信号
- **浪费 time**: 15 分钟整理 + 分类 + 写 message

### 坑 2: 版本号是承诺，不是装饰

- **现象**: curator 本地是 0.4.0，npm 只有 0.1.0，差距 3 个 minor
- **正确做法**: 每次改版本号时对照 semver 规则和 git history 确认合理性
- **根因**: 开发时随手写版本号，没有 publish 前检查 npm 现状
- **浪费 time**: 5 分钟排查
