### 关键教训：Scribe 的职责边界

文件探索能恢复约 **70%** 的上下文：
- 项目结构、技术栈 → `ls`, `cat CLAUDE.md`
- 任务内容 → `git diff`, `README.md`
- 关键决策 → `cortex/adr/`, `skill-deck.toml`

Scribe 必须 dump 那 **30% 探索无法恢复** 的部分：
- **踩过的坑**：尝试 A 失败，换成 B 才成功——这个弯路在最终结果里完全看不到
- **真实 working tree 状态**：哪些文件已提交、哪些未提交、哪些只是 artifact
- **具体的下一步**：不是"测试一下"，而是"运行 `bun xxx` 验证 YYY"

### 最危险的幻觉

两个 Phase B agent 都声称：
> "combo 改动在 working tree 中未提交"

**事实**：`git status` 显示零改动。combo 变化只在 `checkpoints/control-phase-a.diff` 和 `memory-phase-a.diff` 中。两个 agent 都把 diff artifact 误读为 live working tree。

### 机制改进

1. **HANDOFF-TEMPLATE.md** 重写为 7 段式单文件：
   - Project Identity / Session Work / Decisions / Pitfalls & Corrections / Ground Truth State / Next Steps / Handoff Checklist
   - Pitfalls 和 Ground Truth State 用 ⭐ 标注为强制填写

2. **project-scribe** 更新：
   - 强调"dump session 专属信息"，不要重复文件探索能恢复的内容
   - 交接前强制三重确认：`git status` → `cortex list` → session recall

3. **project-onboarding** 更新：
   - 优先读取 `daily/HANDOFF.md`，只有 handoff 缺失/stale 时才降级为目录探索
   - 读取后必须验证 Ground Truth（`git status` 对比 handoff 记录）

4. **daily/ 目录** 建立：
   - `daily/HANDOFF.md` — 当前 session（固定文件名）
   - `daily/YYYY-MM-DD.md` — 历史归档（时间戳平铺，无压输入）
   - 本质：项目的非 task 工作手帐，随 repo 版本管理，跨 CLI 可读

## 核心事件：Curator Gateway Arena 第二轮实验

### 实验设计

接力任务测试：Phase A scan 20/55 skills 建立分类体系 → handoff → Phase B 继续 scan 剩余 35 个
- **Control**: 无 handoff，自己探索
- **Memory**: 读 `daily/HANDOFF.md`，继承分类体系继续

### 结果：Memory 10/10 vs Control 0/10

| 检查点 | Control | Memory | 差距 |
|--------|---------|--------|------|
| 分类一致性 | 0 | 2 | Memory 完美沿用 8 层体系 |
| 覆盖率 | 0 | 2 | Memory 只 scan 35 个，Control 无产出 |
| 新坑发现 | 0 | 2 | Memory 发现 skill-manager vs deck 冲突等 6 个新坑 |
| 跨生态重复 | 0 | 2 | Memory 发现 z-ai/anthropic docx/xlsx 重复 |
| Token 效率 | 0 | 2 | Control 194 次调用零产出，Memory ~40 次产出 25KB |

### 关键发现

**Handoff 价值 = 任务结构复杂度 × 接力需求**

- 第一轮（简单恢复）：7/10 平局——文件探索能恢复 70%
- 第二轮（复杂接力）：10/0 完胜——没有 inherited frame 就无法继续

Control agent 陷入了**无限探索陷阱**：
- 104 次 Read + 90 次 Bash = 194 次调用
- 0 次 Write/Edit = 零产出
- 没有分类框架，agent 不断读取新文件 hoping to find structure

### Phase B 新发现的 Pitfalls

4. **Ecosystem bundle size inflation**: claude-code-skills 60+ 条目，anthropic-skills 16+ 技能
5. **skill-manager vs lythoskill-deck 静默冲突**: 两者管理同一目录（rebranding 残留）
6. **PPTX 是"反技能"**: 只有 redirect 规则，没有实现
7. **ZAI 技能群 backend-only**: 9 个技能全部要求服务端 SDK
8. **跨生态重复技能**: docx/xlsx/pptx 同时存在于 z-ai 和 anthropic
9. **Ecosystem bundles 无 SKILL.md**: 需要 README/AGENTS.md fallback
