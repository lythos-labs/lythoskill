## 3. 踩过的坑与修正

### 坑 1: `parseAgentMd` 硬编码 GitHub 路径
- **错误**: Given 解析默认 `path: github.com/foo/bar/${alias}`，但 refresh/prune 需要 localhost
- **正确**: 支持 `skill-a (localhost)` 语法，path 变为 `localhost/skill-a`
- **根因**: `.agent.md` 的 Given 不够表达力，path 语义隐藏在代码里

### 坑 2: `findSource` 不支持 `localhost/` 前缀
- **错误**: `findSource` 的 FQ 正则 `^[a-z0-9-]+\.[a-z0-9-]+\/` 要求 `.`，`localhost` 不匹配；直接路径是 `coldPool/localhost/skill-a`；扁平扫描用 full name
- **正确**: 新增 `localhost/` 分支，直接映射到 `coldPool/skill-a`
- **根因**: `refresh` 的 skip 逻辑检查了 `localhost`，但 `findSource` 找不到 localhost 文件，两边语义不一致

### 坑 3: `scanColdPoolRepos` 只遍历三层（host/owner/repo）
- **错误**: flat skill `cold-pool/skill-a/` 被当成 host 目录，不会检查 `SKILL.md`
- **正确**: 每层都检查 `SKILL.md`，识别 flat skill 和 standalone repo
- **根因**: 假设 cold-pool 始终是 FQ 三层结构，忽略了 localhost flat 布局

### 坑 4: prune timeout 不是 judge 问题，是 Given 数据错误
- **错误**: 一开始怀疑是 LLM judge 导致超时
- **正确**: agent 执行 `./deck link` 时 GitHub 路径找不到 → agent 可能尝试网络修复 → hang
- **根因**: Given 数据错误导致 agent 行为偏离，300s 硬 timeout 才暴露

### 坑 5: remove 场景 LLM judge 返回非 JSON（"API Error"）
- **错误**: judge 输出解析失败，但 automated judge 兜底通过
- **正确**: LLM judge 是 soft gate，失败不 block 测试；非 JSON 输出被记录到 `judge-verdict.json.error`
- **根因**: claude API 偶发错误，judge 解析器需要更鲁棒的 fallback
