## 2. 关键坑

### Pitfall: Bun workspace package re-export `export { x as y } from 'pkg'` 失败

- **Wrong approach**: `export { claudeCliAdapter as claudeAdapter } from '@lythos/agent-adapter'` 在 re-export chain 中报 `SyntaxError: export 'claudeAdapter' not found in './claude'`
- **Fix**: 改为显式 `import { claudeCliAdapter } from '@lythos/agent-adapter'` 再 `export const claudeAdapter = claudeCliAdapter`
- **Root cause**: bun 对 workspace package 的 `export { a as b } from 'workspace-pkg'` 解析有 bug，可能和 side-effect import + 同包内循环引用有关
- **Time wasted**: ~15 min

### Pitfall: `bun test` 路径参数需要 absolute path

- `bun test packages/.../file.test.ts` 被当 filter（bun test v1.3.11 的 `--cwd` 行为）
- `bun test /Users/.../file.test.ts` 绝对路径才正确
- **Time wasted**: ~5 min

## 2. 关键发现

### DeepSeek `-p` 模式 ≠ Agent 模式
`deepseek -p "prompt"` 和 `deepseek exec` 都是 chat completion 模式——不执行工具。模型输出 "I cannot execute commands on your system" 或 XML 格式的 tool call 文本，但不会实际写文件、搜网页、跑 shell。
- 纯文本任务（hello world、cookie recipe）→ 可用，arena stdout 捕获足够
- 文件/搜索/shell 任务 → 需要 `deepseek serve --http` (HTTP API with full tool execution)
- 适配器当前 `-p` 模式标记为 text-only

### bunx 在 TUI 沙箱下报 PermissionDenied
`bunx @lythos/skill-deck link` 在任何目录都失败——`bunx` 需要写 `~/.bun/` 缓存目录，被 TUI 沙箱拦截。
