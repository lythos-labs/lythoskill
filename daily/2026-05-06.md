## 5. Agent Spawn 脆弱性分析（report.md 调研确认）

引入的 `report.md` 系统分析了 `Bun.spawn claude -p` 在 WebSearch/WebFetch 场景下的问题。确认了 ground truth 踩到的坑：

| 问题 | 根因 | 影响 |
|------|------|------|
| stdin pipe flush (ARM64) | Bun spawn 在 ARM64 上 stdin 无法正确 flush 到子进程 | 子进程 hang / "Input must be provided" |
| 嵌套检测 | CLAUDECODE 等 env 未清理 → CLI 拒绝启动 | spawn 立即失败 |
| `-p` deferred tool 死锁 | v2.1.76+ regression，WebSearch/WebFetch/Agent/Skill 间歇性 hang | 0 字节输出，不可靠 |
| dontAsk 模式 WebSearch 被拒 | WebSearch 的 checkPermissions() 返回 passthrough → deny | WebSearch 在 dontAsk 下不可用 |

**推荐方案排序**（报告第 1.3 节）：
1. Agent SDK `query()` — 最稳定，完整工具访问
2. PTY + `node-pty` — 完整交互能力
3. Bun spawn + workaround 组合 — 脆弱但可 work

**我们的路径**：
- 短期：修 `buildClaudeCommand`（清 env、stdin ignore、prompt file、`--output-format json`、重试包装）
- 中期：加 `AgentSdkAdapter`（spawn → `query()`），AgentAdapter 接口已就位
- 备选：MCP server 模式（用户提及）
