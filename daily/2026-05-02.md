### 关键上下文（文件探索无法恢复）

1. **核心产出：`deck_skill_type` 自定义 frontmatter 字段**
   - 解决 `type` 被 Kimi CLI 硬约束为 `standard|flow` 的问题
   - `deck_skill_type` 用于单卡自我标识：**combo** / **transient** / **fork**
   - **combo**: 携带 `deck_delegates` 路由表，内在架构模式
   - **transient**: 临时 workaround，有过期时间，设计目标"越来越薄"
   - **fork**: 对外部 skill 的本地改编（context-specific，无法 upstream）
   - innate / tool 仍由 `skill-deck.toml` section 决定，不需要单卡声明

2. **Description 写法原则（arena 实证驱动）**
   - hybrid（平静语气 + "Use this when:" trigger 列表）是默认
   - pushy 是 fallback（hybrid 触发不够时退到 pushy）
   - keyword-rich 永远不用（导致幻觉）
   - **"口嫌体正直"**: agent 自我评价中讨厌 pushy，但触发测试中 pushy 最快

3. **Coach 规则积累方法论**
   - 每条规则必须标注：来源、日期、验证状态、置信度
   - **没有来源 = 没有规则 = 可信度不高**
   - 单条错误规则会通过 coach 评审级联到整个 repo 的所有 skill
   - 外部"官方推荐"未经验证不得纳入 coach（如"Claude 推荐 pushy"找不到官方文档，全是第三方二手转述）

4. **GEO = Generative Engine Optimization**
   - skill desc 生态 = web SEO / PageRank 演化的 replay
   - arena 是 PageRank for agents，curator 是 crawler，coach 是 SEO 指南
   - 外部无 skin in the game 的推荐 = "体面话"，arena 实测才是结论

5. **Combo / transient / fork 是 localhost-first**
   - 从项目-specific 实战经验中浮现/铸造，不是上游拉取的
   - `deck update` 跳过 `localhost/*` 是正确设计
   - fork 无法 upstream 是因为无法证明"只在我这里不工作"（context-specific）

### Pitfall 记录

- **"官方推荐"陷阱**: 搜索"Claude 官方 desc 推荐"找不到 docs.anthropic.com 的直接来源，全是第三方 skill marketplace（mcpmarket.com、toolhunter.cc 等）。ADR 中引用的"Claude 0.1 官方推荐 pushy"可能是二手转述。
- **外部搜索 ≠ 验证**: Web search 只能提供待验证假设，结论必须由 arena subagent 测试产出。
- **coach blast radius**: 写进 coach/references/ 的错误规则会被 agent 用来评审所有 skill，错误会自我复制扩散到 README、wiki、ADR。
