## Cold-pool snapshot vs sync dual-mode (ADR Proposed)

讨论"不同项目对版本要求不一样"问题，结论是 **dual-mode**：

```
deck add <locator>          # 默认 snapshot（安全，项目隔离）
deck add <locator> --sync   # symlink 模式（始终最新）
deck sync <name>            # snapshot → sync（显式切换）
deck freeze <name>          # sync → snapshot（pin 版本）
```

**核心设计**：
- Snapshot 默认：新用户不担心上游更新破坏，metadata 记录 `git_head_ref` + `mode: snapshot`
- Sync 可选：想统一更新的用户显式 opt in
- `deck refresh` 对 snapshot 只 report、不操作："3 commits behind. `deck sync` to update."
- Ref-counting 两种模式都适用

ADR: `cortex/adr/01-proposed/ADR-20260507190157540`

